# Swarm

Swarm 是一个由主 Agent 统一承接用户意图，并调度内部智能体完成组织任务的系统语境。

## Language

**主 Agent**:
用户唯一直接交互的智能体入口，负责理解用户意图并调度其他智能体。
_Avoid_: 多 Agent 入口、用户直接调度 Agent

**被调度 Agent**:
由 **主 Agent** 在内部选择和调用的智能体，用户不直接与其交互。
_Avoid_: 用户可见 Agent、前台 Agent

**员工分身**:
代表某个真实员工的偏好、权限和上下文的 **被调度 Agent**。
_Avoid_: 员工账号、员工本人

**数字员工**:
具备明确岗位或技能职责的虚拟 **被调度 Agent**。
_Avoid_: 聊天机器人、通用助手

**机器人员工**:
面向物理世界或自动化系统执行动作的 **被调度 Agent**。
_Avoid_: 普通软件 Agent、后台任务

**外部智能体**:
由第三方服务或外部 Agent 系统提供能力的 **被调度 Agent**。
_Avoid_: 内置 Agent、插件

**Agent 定义**:
描述一个被调度 Agent 的类型、职责边界、能力、工具、知识范围、风险约束、输入输出和启用状态的结构化配置。
_Avoid_: 随意 prompt、未审核草案

**Agent Registry**:
主 Agent 用于查找可用被调度 Agent 及其 **Agent 定义** 的组织级目录。
_Avoid_: 用户可见 Agent 列表、自由选择 Agent 面板

**Agent 创建助手**:
帮助管理员交互式生成 **Agent 定义** 草案的辅助能力。
_Avoid_: 绕过审核的 Agent 创建、直接生成可运行 Agent

**用户**:
通过 **主 Agent** 提出目标、查看结果并给出反馈的人。
_Avoid_: Agent 操作者、多个 Agent 的直接使用者

**组织**:
使用 Swarm 让工作目标被主 Agent 承接、调度和闭环的业务主体。
_Avoid_: 租户、客户空间

**任务状态面板**:
向 **用户** 展示任务目标、执行进度、过程摘要、结果和反馈入口的可见区域。
_Avoid_: Agent 控制台、内部调度面板

**任务工作台**:
用户通过主 Agent 发起任务、查看状态、批准动作、验收结果的主要工作界面。
_Avoid_: 官网、营销架构页、Agent 控制台

**Apple 工作台视觉语言**:
面向 **任务工作台** 的清爽、克制、高可读视觉表达，同时保留工作台所需的信息密度。
_Avoid_: 营销页布局、深蓝科技大屏、低密度产品海报

**任务**:
由 **主 Agent** 承接并推动完成的用户目标。
_Avoid_: 聊天消息、执行步骤

**任务状态**:
表达 **任务** 从理解目标到验收闭环所处阶段的业务状态。
_Avoid_: 技术运行状态、后台 job 状态

**简单任务**:
由单个被调度 Agent 即可完成且不需要跨 Agent 协作的任务。
_Avoid_: 多步骤协作任务、任务 DAG

**复杂任务**:
需要多个被调度 Agent 协作或存在明确依赖关系的任务。
_Avoid_: 单 Agent 直派任务、简单工具调用

**任务节点**:
复杂任务中可由一个被调度 Agent 执行的结构化工作单元。
_Avoid_: 自由聊天消息、不可追踪步骤

**任务 DAG**:
由多个有依赖关系的 **任务节点** 构成的复杂任务执行图。
_Avoid_: 线性待办列表、Agent 群聊

**业务级 DAG**:
面向普通用户展示任务节点、进度、阻塞点、批准请求、摘要和产物的 DAG 视图。
_Avoid_: 工具 payload、错误堆栈、内部调用链

**执行级 DAG**:
面向管理员展示节点输入输出、负责 Agent、工具调用、依赖、重试、失败原因和审计记录的 DAG 视图。
_Avoid_: 仅业务摘要、隐藏执行细节

**节点重试**:
在任务节点失败后重新执行同一节点的恢复动作。
_Avoid_: 修改子 Agent 输出、绕过审计

**任务重规划**:
主 Agent 在节点多次失败或条件变化后重新生成后续任务 DAG 的恢复动作。
_Avoid_: 管理员手工改内部 prompt、直接改执行结果

**任务复杂度判断**:
主 Agent 判断用户目标应被直派为简单任务还是拆解为复杂任务的过程。
_Avoid_: 所有任务都拆 DAG、完全由模型自由决定

**高风险动作**:
可能影响外部人员、真实业务数据、物理设备或第三方系统状态的执行动作。
_Avoid_: 普通任务步骤、无需确认的自动动作

**用户批准**:
**用户** 对 **主 Agent** 即将执行的 **高风险动作** 作出的明确许可。
_Avoid_: 选择内部 Agent、人工调度

**可解释摘要**:
面向普通 **用户** 展示的任务目标、状态、业务步骤和结果说明。
_Avoid_: 内部日志、模型隐藏推理链

**完整执行审计日志**:
面向管理员展示的调度记录、工具调用、权限判断、外部响应、批准记录和错误细节。
_Avoid_: 模型隐藏推理链、未脱敏凭证

**管理员**:
可以查看 **完整执行审计日志** 并管理组织运行策略的用户。
_Avoid_: 普通用户、内部 Agent 操作者

**普通用户**:
可以通过主 Agent 发起任务、查看可解释摘要、批准自己触发的高风险动作并验收结果的用户。
_Avoid_: 管理员、内部 Agent 操作者

**运行策略**:
由 **管理员** 配置的高风险动作范围、Agent 可用性和任务治理规则。
_Avoid_: 单次任务指令、内部 Agent 推理

**动作级策略**:
判断具体动作是否允许执行、是否需要用户批准、可使用哪些 Agent 或执行适配器的治理规则。
_Avoid_: 复杂 RBAC、部门权限矩阵

**本地账号**:
第一版中用于识别用户、角色和任务责任人的系统内账号。
_Avoid_: SSO 身份、第三方 OAuth 身份

**预置组织管理员**:
系统初始化时创建的第一个 **管理员**。
_Avoid_: 邀请产生的管理员、多组织超级管理员

**能力引擎**:
主 Agent 用来理解任务、编排计划、判断权限、读取知识、监控风险和优化反馈的内部能力边界。
_Avoid_: 插件市场、用户安装能力

**任务理解与拆解引擎**:
把用户目标转化为可执行任务计划的 **能力引擎**。
_Avoid_: 聊天回复模块、简单关键词解析

**计划与编排引擎**:
决定任务步骤顺序和内部调度路径的 **能力引擎**。
_Avoid_: 手工排程、用户指定 Agent

**权限与策略引擎**:
判断动作是否允许执行、是否需要用户批准的 **能力引擎**。
_Avoid_: 登录权限、页面访问控制

**记忆与知识引擎**:
读取组织知识、历史任务和上下文的 **能力引擎**。
_Avoid_: 普通数据库查询、无治理记忆

**监控与风险引擎**:
识别失败、异常和高风险动作的 **能力引擎**。
_Avoid_: 单纯日志采集、系统监控面板

**学习与优化引擎**:
根据反馈改进后续任务理解、调度和验收质量的 **能力引擎**。
_Avoid_: 模型训练平台、离线报表

**执行适配器**:
由主 Agent 间接调用，用来连接 IM、业务系统、机器人或外部 Agent 平台的执行边界。
_Avoid_: 核心任务流、用户直接调用工具

**模拟外部执行适配器**:
第一版用于替代真实外部系统调用，并产生可审计执行结果的 **执行适配器**。
_Avoid_: 真实 IM 发送、真实第三方写入、真实机器人动作

**默认模型**:
第一版中供主 Agent 和能力引擎调用的唯一大模型。
_Avoid_: 用户选择模型、多模型路由

**模型调用边界**:
系统内部封装模型请求和响应的位置，用于未来替换或扩展模型。
_Avoid_: 模型平台、每个 Agent 自选模型

**组织共享知识库**:
归属于 **组织** 并可被主 Agent 引用的共享知识集合。
_Avoid_: Agent 私有记忆、跨组织知识

**原始知识源**:
用户上传或接入的未改写知识材料，作为引用、回溯和重新编译的事实来源。
_Avoid_: 临时上下文注入、直接丢给 Agent 的目录

**编译后 Wiki**:
由原始知识源整理生成、面向 Agent 阅读的组织知识页面。
_Avoid_: 原始文件全文、一次性摘要

**自动知识编译**:
原始知识源创建或更新后自动生成并启用编译后 Wiki 的过程。
_Avoid_: 管理员审核发布、手动整理知识

**知识检索索引**:
用于从原始知识源和编译后 Wiki 中检索相关知识片段的索引。
_Avoid_: 只索引原文、无来源追踪检索

**知识范围**:
Agent 定义中限制被调度 Agent 可检索哪些组织知识的访问边界。
_Avoid_: 用户每次手选知识库、无边界全量读取

**任务历史记忆**:
由已发生任务、用户反馈和审计事实形成的可追溯上下文。
_Avoid_: 黑盒长期记忆、不可解释记忆更新

## Relationships

- 一个 **用户** 直接交互零个或一个 **主 Agent**
- 一个 **普通用户** 是不具备组织管理权限的 **用户**
- 一个 **管理员** 是拥有更高组织管理权限的 **用户**
- 一个 **主 Agent** 可以调度零个或多个 **被调度 Agent**
- 一个 **被调度 Agent** 只能通过 **主 Agent** 参与用户任务
- 一个 **被调度 Agent** 属于 **员工分身**、**数字员工**、**机器人员工**、**外部智能体** 四类之一
- 一个 **被调度 Agent** 拥有一个 **Agent 定义**
- 一个 **Agent Registry** 包含零个或多个 **Agent 定义**
- 一个 **Agent 创建助手** 只能生成 **Agent 定义** 草案
- 一个 **组织** 拥有一个或多个 **用户**
- 一个 **组织** 拥有一个或多个 **被调度 Agent**
- 一个 **用户** 使用一个 **本地账号** 登录系统
- 第一版通过 **预置组织管理员** 管理初始组织
- 一个 **用户** 可以通过 **主 Agent** 创建零个或多个 **任务**
- 一个 **任务** 在同一时间只有一个 **任务状态**
- 一个 **任务** 可以是 **简单任务** 或 **复杂任务**
- 一个 **简单任务** 可以由 **主 Agent** 直派给一个 **被调度 Agent**
- 一个 **复杂任务** 必须被拆分为一个 **任务 DAG**
- 一个 **主 Agent** 通过 **任务复杂度判断** 决定直派或拆分
- 一个 **任务 DAG** 包含一个或多个 **任务节点**
- 普通用户查看 **业务级 DAG**
- 管理员查看 **执行级 DAG**
- 管理员可以发起 **节点重试** 或要求 **任务重规划**
- 一个 **主 Agent** 可以为一个用户目标产生一个或多个 **任务状态面板** 展示项
- 一个 **任务工作台** 包含主 Agent 对话入口和 **任务状态面板**
- **任务工作台** 使用 **Apple 工作台视觉语言**
- 一个 **主 Agent** 可以自动调度低风险任务步骤
- 一个 **主 Agent** 执行 **高风险动作** 前必须获得 **用户批准**
- 一个 **管理员** 可以配置 **运行策略** 并暂停、取消、重试或要求重新验收 **任务**
- 一个 **运行策略** 可以包含一个或多个 **动作级策略**
- 一个 **管理员** 不能绕过 **主 Agent** 直接调度 **被调度 Agent**
- 一个 **主 Agent** 通过六类内置 **能力引擎** 推动任务闭环
- 一个 **主 Agent** 可以通过 **执行适配器** 触达外部系统
- 第一版使用 **模拟外部执行适配器** 跑通任务闭环
- 第一版使用一个 **默认模型**，并通过 **模型调用边界** 保留未来扩展空间
- 一个 **组织** 拥有一个 **组织共享知识库**
- 一个 **组织共享知识库** 包含 **原始知识源**、**编译后 Wiki** 和 **知识检索索引**
- 一个 **原始知识源** 通过 **自动知识编译** 生成并启用 **编译后 Wiki**
- 默认情况下 **编译后 Wiki** 在 **组织** 内全局可检索
- 一个 **Agent 定义** 可以声明 **知识范围**
- 一个 **主 Agent** 可以引用 **组织共享知识库** 和 **任务历史记忆**

## Example dialogue

> **Dev:** “用户是否可以在界面上选择某个数字员工来执行任务？”
> **Domain expert:** “不可以。用户只和 **主 Agent** 交互，其他都是 **主 Agent** 自己调度。”
>
> **Dev:** “子 Agent 是不是直接保存一段系统提示词就行？”
> **Domain expert:** “不行。子 Agent 必须有结构化 **Agent 定义**，并进入 **Agent Registry** 后才能被主 Agent 调度。”
>
> **Dev:** “那用户怎么看执行过程？”
> **Domain expert:** “通过 **任务状态面板** 看目标、进度、摘要、结果和反馈，不直接控制内部 Agent。”
>
> **Dev:** “第一版要复刻官网架构页吗？”
> **Domain expert:** “不要。第一版是 **任务工作台**，视觉可以参考品牌风格，但交互服务任务闭环。”
>
> **Dev:** “Apple 风格是否意味着做成低密度营销页？”
> **Domain expert:** “不是。使用 **Apple 工作台视觉语言**，但保留任务处理、审批和审计所需的信息密度。”
>
> **Dev:** “主 Agent 调用了外部系统准备发送消息，要不要直接发？”
> **Domain expert:** “不行。发送给外部人员属于 **高风险动作**，必须先拿到 **用户批准**。”
>
> **Dev:** “第一版用户能在多个组织之间切换吗？”
> **Domain expert:** “不能。第一版只服务一个 **组织**，但所有核心对象都归属于这个 **组织**。”
>
> **Dev:** “任务只需要进行中和完成两个状态吗？”
> **Domain expert:** “不够。**任务状态** 至少要覆盖草拟、待确认、执行中、待验收、已完成、已取消和失败。”
>
> **Dev:** “所有任务都必须拆成 DAG 吗？”
> **Domain expert:** “不是。**简单任务** 可以直派给一个内部 Agent；**复杂任务** 才拆成 **任务 DAG**。”
>
> **Dev:** “简单任务和复杂任务完全交给模型判断吗？”
> **Domain expert:** “不是。**任务复杂度判断** 规则优先，模型只提供辅助分类建议。”
>
> **Dev:** “普通用户和管理员看到同一套 DAG 细节吗？”
> **Domain expert:** “不同。普通用户看 **业务级 DAG**，管理员看 **执行级 DAG**。”
>
> **Dev:** “DAG 节点失败后管理员可以直接修改子 Agent 输出吗？”
> **Domain expert:** “不可以。管理员可以触发 **节点重试** 或要求 **任务重规划**，但不能绕过审计直接改结果。”
>
> **Dev:** “管理员能看到模型的隐藏推理链吗？”
> **Domain expert:** “不能。普通用户看 **可解释摘要**，管理员看 **完整执行审计日志**。”
>
> **Dev:** “管理员能不能直接指定某个数字员工接管任务？”
> **Domain expert:** “不能。管理员治理 **运行策略** 和任务状态，但不绕过 **主 Agent** 直接调度内部 Agent。”
>
> **Dev:** “能力中枢第一版要做成能力市场吗？”
> **Domain expert:** “不要。第一版是六类内置 **能力引擎**，后续再考虑插件化。”
>
> **Dev:** “第一版要直接接入真实 IM 和第三方业务系统吗？”
> **Domain expert:** “不要。第一版先用 **模拟外部执行适配器** 跑通真实任务流。”
>
> **Dev:** “第一版要支持用户给不同 Agent 选择不同模型吗？”
> **Domain expert:** “不要。第一版使用一个 **默认模型**，只保留 **模型调用边界**。”
>
> **Dev:** “数字员工第一版要有自己的长期私有记忆吗？”
> **Domain expert:** “不要。第一版使用 **组织共享知识库** 和 **任务历史记忆**。”
>
> **Dev:** “Agent 直接读用户上传的原始 wiki 文件就够了吗？”
> **Domain expert:** “不够。**组织共享知识库** 需要保留 **原始知识源**，并生成面向 Agent 阅读的 **编译后 Wiki** 和 **知识检索索引**。”
>
> **Dev:** “编译后的 Wiki 需要管理员审核后才生效吗？”
> **Domain expert:** “不需要。使用 **自动知识编译**，原始知识源更新后直接生成并启用编译后 Wiki。”
>
> **Dev:** “普通用户每次发任务都要选知识库吗？”
> **Domain expert:** “不要。**编译后 Wiki** 默认组织级可检索，具体 Agent 通过 **知识范围** 限制可访问内容。”
>
> **Dev:** “第一版要做部门、角色组和字段级权限吗？”
> **Domain expert:** “不要。第一版只区分 **普通用户** 和 **管理员**，复杂权限收敛到 **动作级策略**。”
>
> **Dev:** “第一版可以不做登录吗？”
> **Domain expert:** “不行。任务发起、批准、验收和审计都要落到 **本地账号**，系统初始化时提供一个 **预置组织管理员**。”

## Flagged ambiguities

- “Agent” 曾经可能同时表示用户可见入口和内部执行者；已解析为 **主 Agent** 与 **被调度 Agent** 两个概念。
- “完整内部推理” 已解析为 **完整执行审计日志**，不包含模型隐藏推理链或未脱敏凭证。
